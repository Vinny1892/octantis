## 1. Context

- **Product / System**: Octantis — an AI-powered infrastructure monitoring agent that autonomously investigates incidents via MCP (Model Context Protocol) servers.

- **Current State**: Octantis is a monolithic Python application licensed under MIT. All components — notifiers (Slack, Discord), MCP connectors (Grafana, K8s), pipeline processors (trigger filter, cooldown), and the LangGraph workflow — are tightly coupled in `src/octantis/`. There are 98 tests, a Helm chart with 5 toggleable subcharts, and a Docker image published to ghcr.io. The business model document (v2.0, April 2026) defines an open-core strategy with 7 paid features, SaaS tiers ($299-$3,500+/mo), and self-hosted licenses ($9,600-$60,000+/yr) — but there is no mechanism to separate free from paid functionality.

- **Problem**: Every feature added to the current codebase is immediately available to anyone who clones the repo. This makes monetization impossible — the business model defines paid features (PagerDuty integration, Web UI, Investigation History, Persistent State, Multi-Cluster) but there is no architectural boundary to gate them. The MIT license compounds this: any competitor can fork Octantis, build a SaaS offering on top, and contribute nothing back. Without a plugin system, the project cannot transition from open-source project to sustainable open-core business. The competitive landscape (PagerDuty, Datadog AI, BigPanda, Shoreline.io) demands this transition before Octantis reaches market adoption — retrofitting a plugin system after users depend on a monolithic architecture would be significantly harder and risk breaking changes.

## 3. Proposed Solution

### Overview

Introduce a plugin architecture that makes Octantis extensible across 5 plugin types: Notifiers, MCP Connectors, Storage Backends, Pipeline Processors, and UI Providers. The core Octantis agent ships with built-in plugins (Slack, Discord, Grafana MCP) and defines interface contracts that premium plugins implement. A Plan Gating Engine controls which plugins and how many can be active simultaneously based on the license tier (free, pro, enterprise). The repository license migrates from MIT to AGPL-3.0 to protect against SaaS competitors while keeping the core open-source.

Existing components (Slack notifier, Discord notifier, Grafana MCP connector, K8s MCP connector, trigger filter, cooldown) are refactored into built-in plugins that implement the new interfaces — proving the architecture works before any premium plugins are built.

### Key Decisions

1. **AGPL-3.0 for core, proprietary for premium plugins** — AGPL protects against SaaS parasites (anyone offering Octantis as a service must open-source their modifications). Premium plugins live in a separate private repository with a proprietary license. This is the Grafana model: Grafana core is AGPL, Grafana Enterprise plugins are proprietary.

2. **Python entry points for plugin discovery** — Plugins are discovered via Python `entry_points` (setuptools/pkg_resources), the same mechanism pytest, tox, and Ansible use for plugins. A plugin is a Python package that declares entry points in its `pyproject.toml`. This avoids custom module loading, works with pip/uv, and is familiar to the Python ecosystem.

3. **Interface contracts via Python Protocols** — Each plugin type has a `Protocol` class (PEP 544) defining the required methods. Plugins implement the protocol without inheriting from a base class. This enables structural subtyping (duck typing with type safety) and keeps plugins decoupled from the core.

4. **Plan gating at the registry level** — The Plugin Registry checks the active license/plan before loading a plugin. Free tier: 1 MCP connector, built-in notifiers only. Pro: up to 3 MCP connectors, enterprise notifiers. Enterprise: unlimited. The gate is enforced at plugin load time, not at call time — failing fast with clear error messages.

5. **MCP slot model for plan differentiation** — Free users can use any single MCP server (Grafana, K8s, Docker, AWS — their choice). Paid users get multiple simultaneous MCP connections. The differentiator is quantity, not which MCPs are available. This avoids fragmenting the MCP ecosystem and keeps all connector code open-source.

### Out of Scope

- **Specific premium plugin implementations** (PagerDuty, Opsgenie, Teams, Email, Web UI, Postgres storage, Investigation History, Pattern Learning) — Each will be a separate PRD built on top of this plugin architecture
- **SaaS platform / multi-tenant infrastructure** — Covered by a future PRD for Octantis Cloud
- **Billing / payment processing** — Requires SaaS platform; this PRD only defines the plan gating mechanism
- **Plugin marketplace / registry service** — Q2 2027 per roadmap; this PRD covers local plugin discovery only
- **CLA (Contributor License Agreement)** — May be needed for AGPL; deferred to legal review

## 8. Risks

| Risk | Impact | Mitigation |
|------|--------|------------|
| Plugin refactoring introduces regressions in existing functionality | High — breaks production for current users | Refactor one component type at a time (notifiers first, then MCPs, then pipeline). Run full test suite after each. No behavior changes — only structural. |
| AGPL-3.0 scares away potential contributors | Medium — fewer community contributions | Clear LICENSING.md explaining that AGPL only affects SaaS redistribution, not self-hosted use. Contributing guide explains CLA is not required (AGPL is copyleft, contributions are automatically AGPL). Grafana, MongoDB, and MinIO thrive under AGPL. |
| Plugin interface contracts lock in a bad API surface early | High — breaking changes are costly once external plugins exist | Start with `0.x` SDK version, document that interfaces may change until `1.0`. Only stabilize after at least 3 plugin types are proven in production. |
| License key bypass is trivial (open-source code) | Medium — tech-savvy users skip paying | Accept this — open-core model relies on honest enterprises, not DRM. Premium plugins in private repos provide stronger protection. Plan gating is a convenience barrier, not a security barrier. |
| Third-party dependency is AGPL-incompatible | Medium — blocks license migration | Run `pip-licenses` audit before migration. Replace any incompatible dependency. |
| Performance overhead from plugin indirection | Low — Python method dispatch overhead is negligible | Benchmark before/after refactoring. Plugin `setup()` runs once at startup; hot path calls (notify, investigate) are direct method calls on already-loaded plugin instances. |

## 10. Architecture Overview

```
                        Octantis Core (AGPL-3.0)
    ┌──────────────────────────────────────────────────────┐
    │                                                      │
    │   ┌──────────────┐    ┌─────────────────────────┐    │
    │   │ Plan Gating  │    │    Plugin Registry      │    │
    │   │ Engine       │◄──►│  (discovery + lifecycle) │    │
    │   │ free/pro/ent │    └────────┬────────────────┘    │
    │   └──────────────┘             │                     │
    │                     ┌──────────┼──────────┐          │
    │                     ▼          ▼          ▼          │
    │              ┌──────────┐ ┌────────┐ ┌────────┐     │
    │              │Notifiers │ │  MCP   │ │Storage │     │
    │              │Protocol  │ │Protocol│ │Protocol│     │
    │              └────┬─────┘ └───┬────┘ └───┬────┘     │
    │                   │           │           │          │
    │   Built-in:       │           │           │          │
    │   ┌───────┐ ┌─────┤     ┌─────┤     ┌────┤          │
    │   │Slack  │ │Discord    │Grafana    │Memory│         │
    │   └───────┘ └─────┘     │K8s  │     └─────┘         │
    │                         └─────┘                      │
    └──────────────────────────────────────────────────────┘

                    Premium Plugins (Proprietary, private repo)
    ┌──────────────────────────────────────────────────────┐
    │  PagerDuty │ Opsgenie │ Postgres │ Web UI │ ...     │
    │  (notifier)│(notifier)│(storage) │(ui)    │         │
    └──────────────────────────────────────────────────────┘

    Plan Gating:
      free:       1 MCP slot,  built-in notifiers, in-memory storage
      pro:        3 MCP slots, all notifiers, persistent storage
      enterprise: unlimited,   all plugins, UI providers
```

- **Plugin Registry**: Discovers plugins via Python entry points, validates Protocol compliance, manages lifecycle (setup/teardown), exposes `get_plugin()` and `list_plugins()` APIs
- **Plan Gating Engine**: Reads license key (env var or file), determines tier (free/pro/enterprise), enforces slot limits and plugin type access at load time
- **Plugin Protocols**: Five `typing.Protocol` interfaces (`NotifierPlugin`, `MCPConnectorPlugin`, `StoragePlugin`, `PipelineProcessorPlugin`, `UIProviderPlugin`) defining the contract each plugin type must satisfy
- **Built-in Plugins**: Existing components (Slack, Discord, Grafana MCP, K8s MCP, trigger filter, cooldown, in-memory state) refactored to implement Protocols and registered via entry points
- **Premium Plugins**: Future paid plugins in a private repository, installed via pip from a private registry, discovered by the same entry point mechanism

## 11. Decision Log

| Date | Decision | Rationale |
|------|----------|-----------|
| 2026-04-10 | AGPL-3.0 for core repository | Protects against SaaS competitors forking without contributing back. Grafana built a $6B+ business on this model. MIT provides zero protection. BSL (HashiCorp) and SSPL (MongoDB) are too restrictive and not OSI-approved. |
| 2026-04-10 | Python entry points for plugin discovery | Standard Python packaging mechanism, familiar to ecosystem (pytest, tox, Ansible use it). Avoids custom module loading, directory scanning, or import hacks. Works with pip, uv, and all package managers. |
| 2026-04-10 | Protocol-based interfaces (PEP 544) over ABC inheritance | Structural subtyping keeps plugins decoupled from Octantis internals. Plugin doesn't need to import or inherit from core classes. Enables type checking without runtime dependency. |
| 2026-04-10 | MCP slot model (quantity gating) over feature gating | All MCP connector code stays open-source. Differentiator is how many MCPs run simultaneously, not which ones. Avoids fragmenting the connector ecosystem and keeps contributions flowing to all connectors. |
| 2026-04-10 | Plan gating at load time, not call time | Fail fast with clear error at startup rather than silent failures during incident processing. An operator knows immediately if they need to upgrade, not when a critical alert is missed at 3 AM. |
